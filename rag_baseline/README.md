# Flat RAG Baseline

Ablation study counterpart to the **Graph RAG** pipeline (`pipeline.py`).  
Both approaches share the same LLMs, the same knowledge content, and the same
four-stage architecture. **Only the retrieval mechanism differs**, isolating
the contribution of structured graph retrieval versus flat cosine similarity.

---

## Design Principle (fair comparison)

| Stage | Graph RAG (`pipeline.py`) | Flat RAG (`pipeline_rag.py`) |
|---|---|---|
| **1 · Classify** | LLM → pattern + keywords (structured output) | **identical** |
| **2 · Retrieve** | Neo4j graph: typed nodes + relations (`USES_OPERATOR`, `INSTANCE_OF`) | Cosine similarity over embedded text chunks |
| **3 · Generate** | LLM with template + operator rules + examples | LLM with top-k mixed chunks |
| **4 · Validate** | ANTLR parser service | **identical** |

The same 28 patterns, operators, and examples from `seed_kg.py` are used in
both approaches — in Graph RAG they live in Neo4j; here they are embedded with
`text-embedding-3-small` and stored as numpy arrays.

## Setup

```bash
# 1. Install dependencies (already in requirements.txt)
pip install openai numpy

# 2. Build the vector index (run once)
cd rag_baseline
python build_index.py
```

## Run a single constraint

```bash
python pipeline_rag.py "Drone returns to base when battery < 10%"
python pipeline_rag.py "Robot must stop immediately when obstacle detected" --k 15
```

## Run the full benchmark

```bash
# Benchmark all 40 scenarios across all models
python benchmark_rag.py --folder ../dataset --output rag_benchmarks

# Benchmark a slice
python benchmark_rag.py --folder ../dataset --start 0 --end 10 --output rag_benchmarks
```

Output is written to `rag_benchmarks/<scenario_id>_rag.json` — same schema as
`pipeline_benchmarks/` for direct comparison.

---

## Key expected differences vs Graph RAG

- **Operator selection**: Flat RAG retrieves operators only by similarity, not
  by structural `USES_OPERATOR` links → higher risk of irrelevant operators in
  context.
- **Pattern grounding**: Pattern templates may not be retrieved if the embedding
  similarity is outcompeted by high-scoring examples of a different pattern.
- **No structural filtering**: retrieved chunks are a mixed bag of patterns,
  operators, and examples, regardless of which pattern was classified.

These differences directly align with the failure modes analysed in Section 4.2:
under-specification and incorrect logical structure.
